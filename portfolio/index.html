MAIL CHAT

  • When: June 2024 - July 2024.
  • Where: You can check out the project code here .
  • What: I developed this multi-microservice, highly scalable application as a result of learning a new language — Go. The application enables users to chat in an email-like system.
  • How
    • The app includes five distinct, self-contained microservices: Broker Service, Authentication Service, Listener Service, Logging Service, and a Mail Service.
    • The frontend is built with HTML and styled with Bootstrap for responsiveness, communicating with the backend via the Broker Service.
    • The Broker Service handles events from the frontend and pushes them to RabbitMQ, utilized by the Listener Service for proper distribution.
    • The entire application is containerized with Docker and deployed on Kubernetes and Docker Swarm for scalability

CALENDAR 2.0

  • When: March 2024 - May 2024.
  • Where: Project code posted: frontend and backend .
  • What: This task-management application was developed in collaboration with my upperclassman friend, Binh Tran. It integrates features similar to Apple Calendar, allowing users to create accounts using email and manage tasks on a calendar-like UI.
  • How
    • The backend was built using Express.js for routing, an SQL API for handling tasks, and bcrypt with Mongoose for authentication.
    • The frontend was developed with React to provide real-time event tracking and handle response data from the backend.
    • The application utilizes OracleDB for task management and MongoDB Atlas for user data storage.

BOOK SEARCHER SYSTEM

  • When: December 2023
  • Where: Project code posted: frontend and backend .
  • What: This is a highly scalable backend Distributed System that implements a book-searching UI.
  • How
    • The application uses the TF-IDF algorithm to rank books by their relevance.
    • The backend integrates Apache Zookeeper for high scalability and consistency. It is eligible to detect and recover when one cluster or aggregator fails.
